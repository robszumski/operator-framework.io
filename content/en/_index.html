OperatorFramework

The Operator Framework is an open source toolkit to manage Kubernetes native applications, called Operators, in an effective, automated, and scalable way.

What is an Operator? How can I build one?

What's in the Framework?

The Operator Framework is set of tools to realize the vision of Operators in the Kubernetes ecosystem. Using the Framework should save you time building an Operator by boostrapping your project and it should make it easy to run that Operator on a Kubernetes cluster. These tools don't have to be used together of course, but it's the happy path for getting started and provides the most value to the community as a holistic toolset.

Build

Software development kit for building Kubernetes Operators. All you need to do is bring the business logic that makes your app tick.

The SDK Provides high level APIs, useful abstractions for testing, and project scaffolding. The SDK is available on Github.

Manage

Safely run a fleet of Operators on your Kubernetes clusters. Helps install, update, and manage the lifecycle of Operators running across clusters.

The Operator Lifecycle Manager can navigate the complexities of CRDs, including protecting you from conflicts and restricting Operators to subsets of namespaces. OLM is available on Github.

Discover

A flourishing ecosytem of Operators are ready to run on Kubernetes today. Discover databases, AI/ML services, monitoring, security tools and more on OperatorHub.io.

Open source communities like Prometheus, Rook.io, and Istio are embracing Operators as a way to ease install and management. All Operators are welcome, regardless of if they were built with the SDK or another tool.

What is an Operator?

The goal of an Operator is to put operational knowledge into software. Containerized software runs within environments that are changing every millisecond and the old tools for configuration management and application deployment don't work at this speed and typically require a human in the loop.

Sometimes this knowledge only resided in the minds of administrators or in various combinations of shell scripts. Kubernetes provides the foundation to run scalable and reslient systems, but the application-specific knowledge needs to be taught to the cluster. Operators encapsulate that knowledge and bring a cloud-like ease of use to your Kubernetes clusters.

Operators implement and automate common Day-1 (installation, configuration, etc) and Day-2 (re-configuration, update, backup, failover, restore, etc.) activities in a piece of software running inside your Kubernetes cluster, by marrying Kubernetes primatives and APIs to the business logic of a database or web application. We call this a Kubernetes-native application.

With Operators you can stop treating an application as a collection of primitives like Pods, Deployments, Services or ConfigMaps, but instead as a single object that only exposes the knobs that make sense for the application.
